# converter-for-clunkers.py — Markdown to DOCX converter

Скрипт конвертує `.md` файли лабораторних робіт та РГР у форматовані `.docx` з титульною сторінкою, змістом, нумерацією сторінок та академічним форматуванням.

## Залежності

- Python 3.10+
- [pandoc](https://pandoc.org/) — конвертація Markdown у DOCX
- [python-docx](https://python-docx.readthedocs.io/) — пост-обробка DOCX

```bash
pip install python-docx
```

### Встановлення pandoc (Windows)

1. Завантажити інсталятор з [pandoc.org/installing](https://pandoc.org/installing.html) (файл `pandoc-X.X-windows-x86_64.msi`)

2. Запустити інсталятор. Pandoc автоматично додасться у PATH.

3. Перевірити встановлення — відкрити **cmd** (Пуск -> ввести `cmd` -> Enter) та виконати:
   
   ```
   pandoc --version
   ```
   
   Якщо з'явилась версія — все працює.

4. Якщо команда не знайдена — додати pandoc у PATH вручну:
   
   - Пуск -> ввести `змінні середовища` -> вибрати **"Змінити змінні середовища для системи"**
   - Натиснути **"Змінні середовища..."**
   - У розділі **"Змінні середовища користувача"** знайти змінну `Path` -> **Змінити**
   - Натиснути **Створити** та вставити шлях до теки з pandoc (зазвичай `C:\Users\<ваш_користувач>\AppData\Local\Pandoc\`)
   - Натиснути **OK** у всіх вікнах
   - Закрити та відкрити cmd заново, повторити `pandoc --version`

5. Альтернатива (без зміни PATH) — вказати шлях у конфігу:
   
   ```json
   {
       "pandoc_path": "C:\Users\ім’я_вашого_користувача\AppData\Local\Pandoc\pandoc.exe"
   }
   ```

Скрипт шукає pandoc автоматично: спочатку `pandoc_path` з конфігу, потім у системному PATH.

## Швидкий старт

```bash
# 1. Створити конфіг з дефолтними значеннями
python converter-for-clunkers.py create-config

# 2. Відредагувати конфіг (дисципліна, викладач, студент тощо)

# 3. Сканувати теку — знайти всі .md файли та записати у конфіг
python converter-for-clunkers.py scan

# 4. Конвертувати всі файли з конфігу
python converter-for-clunkers.py

# Один файл
python converter-for-clunkers.py -f ЛБ1_Ініціація_проекту.md

# Без титульної сторінки
python converter-for-clunkers.py -f file.md -n

# Список файлів
python converter-for-clunkers.py -L list.txt

# Інший конфіг
python converter-for-clunkers.py -c ../other/converter-config.json
```

Результат зберігається у теку `output/` поруч зі скриптом.

## Структура вихідного документа

1. **Титульна сторінка** — ХАІ, дисципліна, тема, студент, викладач, оцінка
2. **Зміст** — автоматичне поле TOC (оновити у Word: ПКМ на змісті -> Оновити поле)
3. **Тіло звіту** — конвертований Markdown

## Команди

### `create-config`

```bash
python converter-for-clunkers.py create-config
python converter-for-clunkers.py create-config -c path/to/config.json
```

Створює `converter-for-clunkers.json` у поточній теці з усіма параметрами за замовчуванням та порожнім масивом `"files"`. Не перезаписує існуючий конфіг — якщо файл вже є, видалите його вручну або вкажіть інший шлях через `-c`.

### `scan`

```bash
python converter-for-clunkers.py scan
python converter-for-clunkers.py scan -c path/to/config.json
```

Сканує робочу теку (та підтеки на 1 рівень вглиб) на `.md` файли, парсить метадані з імен файлів та записує масив `"files"` у конфіг.

Розпізнає префікси:

| Префікс    | Тип роботи                   |
| ---------- | ---------------------------- |
| `ЛБ`, `ЛР` | Лабораторна робота           |
| `РГР`      | Розрахунково-графічна робота |
| `КР`       | Курсова робота               |
| `ПР`       | Практична робота             |

### Формат імені файлу

```
<ПРЕФІКС><номер><роздільник><тема>.md
```

Роздільники між словами теми — `_`, `-` або пробіл (взаємозамінні, послідовні схлопуються). Номер — необов'язковий (для РГР, наприклад).

Приклади:

| Ім'я файлу                 | Тип                          | Номер     | Тема              |
| -------------------------- | ---------------------------- | --------- | ----------------- |
| `ЛБ1_Ініціація_проекту.md` | Лабораторна робота           | 1         | Ініціація проекту |
| `ЛБ1-Ініціація-проекту.md` | Лабораторна робота           | 1         | Ініціація проекту |
| `ЛБ1 Ініціація проекту.md` | Лабораторна робота           | 1         | Ініціація проекту |
| `ЛБ12_Тестування.md`       | Лабораторна робота           | 12        | Тестування        |
| `РГР_Скрам_критика.md`     | Розрахунково-графічна робота | *(немає)* | Скрам критика     |
| `ПР3_Аналіз_вимог.md`      | Практична робота             | 3         | Аналіз вимог      |
| `notes.md`                 | *(не розпізнано)*            |           |                   |

Файли без відомого префіксу потрапляють у конфіг з порожніми метаданими — відредагуйте вручну або видаліть.

Існуючі записи у конфігу зберігаються (метадані не перезаписуються). Нові файли позначаються `*` у виводі. Після сканування відредагуйте конфіг — видаліть зайві файли, поправте теми.

## CLI флаги

| Короткий | Довгий                | Опис                                                             |
| -------- | --------------------- | ---------------------------------------------------------------- |
| `-f`     | `--file`              | Конвертувати один `.md` файл                                     |
| `-L`     | `--list`              | Шлях до файлу зі списком файлів                                  |
| `-n`     | `--no-title`          | Не додавати титульну сторінку                                    |
|          | `--with-title`        | Додати титульну сторінку (скасовує конфіг `"no_title": true`)    |
| `-H`     | `--keep-hr`           | Не видаляти горизонтальні лінії (`---`)                          |
|          | `--no-hr`             | Видаляти горизонтальні лінії (скасовує конфіг `"keep_hr": true`) |
|          | `--no-toc`            | Не додавати зміст                                                |
|          | `--with-toc`          | Додати зміст (скасовує конфіг `"no_toc": true`)                  |
|          | `--no-page-numbers`   | Не нумерувати сторінки                                           |
|          | `--with-page-numbers` | Нумерувати сторінки (скасовує конфіг `"page_numbers": false`)    |
| `-c`     | `--config`            | Шлях до JSON конфігу                                             |
| `-d`     | `--discipline`        | Перевизначити дисципліну                                         |
| `-p`     | `--teacher`           | Перевизначити викладача (`"посада ПІБ"`)                         |
| `-s`     | `--student`           | Перевизначити ПІБ студента                                       |
| `-t`     | `--type`              | Тип роботи (`"Лабораторна робота"`)                              |
| `-N`     | `--num`               | Номер роботи                                                     |
| `-T`     | `--topic`             | Тема роботи                                                      |

### Пріоритет налаштувань

CLI флаги завжди перевизначають конфіг. Порядок:

1. **DEFAULTS** (захардкоджені значення у скрипті)
2. **JSON конфіг** — перезаписує defaults
3. **CLI флаги** — перезаписують конфіг

Boolean прапорці мають пари `--no-X` / `--with-X`, що дозволяють перевизначити конфіг в обидва боки. Наприклад, якщо у конфігу `"no_title": true`, флаг `--with-title` примусово увімкне титульну сторінку.

## Конфігурація (`converter-for-clunkers.json`)

#### Шляхи

| Поле          | Тип    | За замовчуванням | Опис                                                                              |
| ------------- | ------ | ---------------- | --------------------------------------------------------------------------------- |
| `pandoc_path` | string | автопошук у PATH | Повний шлях до `pandoc.exe`. Потрібен лише якщо pandoc не додано у системний PATH |

| Поле                | Тип  | За замовчуванням | Опис                                                                               |
| ------------------- | ---- | ---------------- | ---------------------------------------------------------------------------------- |
| `no_title`          | bool | `false`          | `true` — не генерувати титульну сторінку. Аналог CLI флагу `-n`                    |
| `no_toc`            | bool | `false`          | `true` — не генерувати сторінку зі змістом                                         |

| Поле             | Тип    | За замовчуванням  | Опис                                                             |
| ---------------- | ------ | ----------------- | ---------------------------------------------------------------- |

| `font_size`      | int    | `14`              | Розмір основного шрифту в pt                                     |
| `code_font_name` | string | `Courier New`     | Шрифт для блоків коду (``` у markdown)                           |

| Поле                   | Тип    | За замовчуванням | Опис                                                                           |
| ---------------------- | ------ | ---------------- | ------------------------------------------------------------------------------ |
| `line_spacing`         | float  | `1.08`           | Міжрядковий інтервал. `1.0` — одинарний, `1.5` — полуторний, `2.0` — подвійний |
| `first_line_indent`    | float  | `1.0`            | Відступ першого рядка абзацу в см (абзацний відступ, «червоний рядок»)         |
| `heading_align`        | string | `center`         | Вирівнювання заголовків: `left`, `center` або `right`                          |
| `heading_space_before` | int    | `6`              | Вертикальний відступ перед заголовком в pt                                     |
| `heading_space_after`  | int    | `0`              | Вертикальний відступ після заголовку в pt                                      |

#### Списки

| Поле                  | Тип   | За замовчуванням | Опис                                                                                                                                               |
| --------------------- | ----- | ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `list_left_indent`    | float | `1.0`            | Відступ тексту пунктів списку від лівого поля сторінки в см                                                                                        |
| `list_hanging_indent` | float | `0.5`            | Виступ маркера (•) або номера (1.) вліво від тексту пункту в см. Маркер розміщується на відстані `list_left_indent − list_hanging_indent` від поля |

Візуально:

```
|← поле сторінки →|← 0.5cm →|← текст пункту ─────→|

                              продовження тексту
                   •          Наступний пункт
```

| --------------------- | ----- | ---------------- | -------------------------------------------------- |
| `title_margin_top`    | float | `2.0`            | Верхнє поле титульної сторінки                     |

| `title_margin_left`   | float | `2.0`            | Ліве поле титульної сторінки                       |
| `title_margin_right`  | float | `2.0`            | Праве поле титульної сторінки                      |
| `body_margin_top`     | float | `2.0`            | Верхнє поле основного тексту                       |
| `body_margin_bottom`  | float | `2.0`            | Нижнє поле основного тексту                        |
| `body_margin_left`    | float | `3.0`            | Ліве поле основного тексту (більше для підшивання) |
| `body_margin_right`   | float | `1.5`            | Праве поле основного тексту                        |

#### Список файлів

Масив `"files"` визначає які файли конвертувати при запуску без `-f`/`-L`. Також використовується для пошуку метаданих при `-f`.

```json
"files": [
    {
        "file": "ЛБ1_Ініціація_проекту.md",
        "type": "Лабораторна робота",
        "num": "1",
        "topic": "Ініціація проекту"
    }
]
```

| Поле    | Тип    | Опис                                                                                | Обов'язкове |
| ------- | ------ | ----------------------------------------------------------------------------------- | ----------- |
| `file`  | string | Шлях до `.md` файлу відносно робочої теки                                           | так         |
| `type`  | string | Тип роботи для титулки (напр. `Лабораторна робота`, `Розрахунково-графічна робота`) | ні          |
| `num`   | string | Номер роботи (порожній рядок якщо немає)                                            | ні          |
| `topic` | string | Тема роботи для титулки                                                             | ні          |

Якщо `type`/`num`/`topic` не вказані — файл конвертується без титульної сторінки.

## Конвенції Markdown

### Підписи таблиць

Рядок `Таблиця: опис` перед таблицею автоматично нумерується у форматі `розділ.номер`:

```markdown
### 2. Планування

Таблиця: Розподіл задач

| Задача | Виконавець |
|--------|------------|
| ...    | ...        |

Таблиця: Терміни

| Етап | Дедлайн |
|------|---------|
| ...  | ...     |
```

Результат у DOCX: `Таблиця 2.1 — Розподіл задач`, `Таблиця 2.2 — Терміни` (ліворуч, нежирний, прив'язаний до таблиці, рядки таблиці не розриваються між сторінками).

### Підписи рисунків

Рядок `Рисунок: опис` перед зображенням нумерується аналогічно:

```markdown
### 3. Архітектура

Рисунок: Діаграма компонентів

![](img/components.png)
```

Результат: `Рисунок 3.1 — Діаграма компонентів`.

### Зображення

Стандартний синтаксис Markdown. Шляхи відносно `.md` файлу:

```markdown
![Опис](img/diagram.png)
```

Зображення вбудовуються у `.docx` автоматично.

### Списки

Маркований список (буллетпоінти) — рядки з `-`, `*` або `+`:

```markdown
- перший пункт
- другий пункт
- третій пункт
```

Нумерований список:

```markdown
1. перший пункт
2. другий пункт
3. третій пункт
```

Вкладені списки — додатковий відступ (2 або 4 пробіли):

```markdown
- основний пункт
  - підпункт
  - підпункт
- наступний пункт
  1. нумерований підпункт
  2. нумерований підпункт
```

Відступи списків налаштовуються через `list_left_indent` та `list_hanging_indent` у конфігу.

### Вирівнювання по лівому краю

HTML-коментарі `<!-- left -->` та `<!-- /left -->` задають блок із вирівнюванням ліворуч (замість justify):

```markdown
<!-- left -->
Цей текст буде вирівняно ліворуч.
<!-- /left -->
```

### Блоки коду

Огорнуті у ` ``` ` блоки автоматично отримують:

- вирівнювання ліворуч
- без відступу першого рядка
- шрифт `code_font_name` / `code_font_size` з конфігу

### Заголовки

Перший `#` (заголовок документа) видаляється — він дублює титульну сторінку. Решта заголовків автоматично зсуваються так, щоб найменший рівень став `Heading 1`:

- Якщо файл використовує `##` як секції: `##` → `Heading 1`, `###` → `Heading 2`
- Якщо файл використовує `###` як секції: `###` → `Heading 1`, `####` → `Heading 2`
- Якщо файл використовує `#` як секції: без зсуву

Зсув визначається автоматично — шукається мінімальний рівень заголовку в документі (після видалення першого `#`).

Нумерація таблиць та рисунків прив'язана до `Heading 1`. Якщо заголовок містить номер на початку (напр. `## 2 Планування`), цей номер використовується для нумерації таблиць (`2.1`, `2.2`). Якщо номера немає — використовується автоінкремент.

### Горизонтальні лінії

Рядки `---` видаляються за замовчуванням. Щоб зберегти — `"keep_hr": true` у конфігу або `-H` флаг.

## Файл списку (`list.txt`)

Tab-separated формат для пакетної конвертації через `-L`:

```
# Коментарі починаються з #
ЛБ1_Ініціація_проекту.md    Лабораторна робота    1    Ініціація проекту
ЛБ2_WBS_та_покерне_планування.md    Лабораторна робота    2    WBS та покерне планування
```

Формат: `файл<TAB>тип<TAB>номер<TAB>тема`. Якщо вказано лише ім'я файлу без метаданих, конвертується без титулки.

## Відомі обмеження

- Зміст потребує ручного оновлення у Word (ПКМ -> Оновити поле -> Оновити цілком)
- Word має бути закритий перед повторною конвертацією (інакше `PermissionError`)
- Mermaid-діаграми не конвертуються (pandoc їх не підтримує)
- Титульна сторінка заточена під шаблон ХАІ
